# 微信支付服务

> 本文档定义微信支付集成的完整实现规范，遵循单一职责原则。

## 输入接口

- 用户请求：创建支付订单、查询订单状态
- 微信回调：支付结果通知、退款通知

## 输出接口

- 支付参数：返回给小程序调起支付的参数
- 订单状态：更新数据库订单状态
- 订阅状态：更新用户订阅权益

---

## 支付流程

```
1. 小程序调用 wx.requestPayment() 前的准备工作:
   - 后端调用微信支付统一下单API
   - 生成预支付交易会话标识(prepay_id)
   - 计算签名并返回支付参数

2. 用户完成支付后:
   - 微信服务器回调后端通知接口
   - 后端验证签名，更新订单状态
   - 更新用户订阅状态和点数
```

## API接口

### 创建支付订单

```
POST /api/payment/create
Authorization: Bearer <token>
Content-Type: application/json

Request:

    "order_id": "ORDER_2026031712345678"

Response:
{
    "success": true,
    "data": {

        "nonceStr": "abc123...",
        "package": "prepay_id=wx123...",
        "signType": "RSA",
        "paySign": "签名值..."

```

### 微信支付回调

```
POST /api/payment/callback
Content-Type: application/json
Wechatpay-Signature: 签名
Wechatpay-Timestamp: 时间戳
Wechatpay-Nonce: 随机串

Request (微信服务器发送):
{
    "id": "EV-201802...",
    "create_time": "2026-03-17T10:41:00+08:00",
    "resource_type": "encrypt-resource",
    "event_type": "TRANSACTION.SUCCESS",
    "resource": {
        "algorithm": "AEAD_AES_256_GCM",
        "ciphertext": "加密内容...",
        "nonce": "随机串",
        "associated_data": "transaction"
    }
}

Response:
{
    "code": "SUCCESS",
    "message": "成功"
}
```

## 签名验证流程

```python
# 回调签名验证步骤
1. 从请求头获取:
   - Wechatpay-Signature: Base64编码的签名
   - Wechatpay-Timestamp: 时间戳
   - Wechatpay-Nonce: 随机字符串
   - Wechatpay-Serial: 平台证书序列号

2. 构造验签名串:
   签名串 = 时间戳 + "\n" + 随机串 + "\n" + 请求体 + "\n"

3. 使用微信平台公钥验证签名:
   - 从微信平台证书获取公钥
   - 使用SHA256withRSA验证签名

4. 解密回调数据:
   - 使用API v3密钥解密resource.ciphertext
   - 验证订单金额、商户号等信息
```

## 配置参数

| 配置项 | 环境变量 | 说明 |
|--------|----------|------|
| 商户号 | WECHAT_MCH_ID | 微信支付商户号 |
| API v3密钥 | WECHAT_API_V3_KEY | 用于回调数据解密 |
| 证书序列号 | WECHAT_SERIAL_NO | 商户API证书序列号 |
| 私钥路径 | WECHAT_PRIVATE_KEY | PEM格式私钥文件路径 |

## 安全检查清单

- [ ] 回调签名验证必须通过
- [ ] 订单金额校验（回调金额 == 订单金额）
- [ ] 商户号校验（回调商户号 == 配置商户号）
- [ ] 防止重复处理（检查订单状态是否已处理）
- [ ] 幂等性保证（同一交易号多次回调结果一致）
