# 系统架构

> 本文档定义AI亲音胎教微信小程序的整体系统架构与部署方案。

## 输入接口

- 用户请求：微信小程序前端HTTPS请求
- TTS服务请求：云后端通过FRP隧道调用本地TTS服务
- 微信回调：微信支付、小程序事件通知

## 输出接口

- 小程序前端响应：JSON格式API响应
- 音频文件：OSS签名URL
- 微信支付参数：支付签名数据

---

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      微信小程序前端                          │
└─────────────────────────────────────────────────────────────┘
                            ↓ HTTPS
┌─────────────────────────────────────────────────────────────┐
│                   阿里云服务器 (云后端)                       │
├─────────────────────────────────────────────────────────────┤
│  FastAPI服务  │  MongoDB  │  OSS存储  │  微信支付           │
└─────────────────────────────────────────────────────────────┘
          ↓ FRP隧道                      ↓ API调用
┌──────────────────────────┐    ┌─────────────────────────────┐
│  本地服务器 (16GB VRAM)  │    │      通义千问 API           │
│  vllm-omni + Qwen3-TTS   │    │      内容生成服务           │
└──────────────────────────┘    └─────────────────────────────┘
```

---

## 部署方案

| 服务 | 部署位置 | 说明 |
|------|----------|------|
| 微信小程序 | 微信平台 | 前端应用 |
| 业务后端 | 阿里云 | FastAPI + MongoDB |
| TTS服务 | 本地服务器 | 16GB VRAM, vllm-omni |
| FRP隧道 | 阿里云→本地 | 内网穿透，端口18001 |
| 对象存储 | 阿里云OSS | 音频文件存储 |
| 内容生成 | 云端API | 通义千问 |

---

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | 微信小程序原生开发 |
| 后端框架 | FastAPI (Python 3.10+) |
| 数据库 | MongoDB 6.0 |
| TTS引擎 | vllm-omni + Qwen3-TTS |
| 内容生成 | 通义千问API |
| 对象存储 | 阿里云OSS |
| 内网穿透 | FRP |
| 部署 | 阿里云ECS + 本地服务器 |

---

## 依赖关系

```
微信小程序
    └── 云后端API (FastAPI)
            ├── MongoDB (数据存储)
            ├── Redis (缓存/会话)
            ├── OSS (音频存储)
            ├── 通义千问API (文本生成)
            └── 本地TTS服务 (通过FRP隧道)
                    └── Qwen3-TTS模型
```
